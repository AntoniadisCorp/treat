# Treat Design System Foundations

## Product Direction
This UI system is aligned to the operations dashboard described in the project prompts: scraper operations, schedules, notifications, RBAC, and admin workflows.

## Visual Principles
- Stable and operational, not startup-generic
- High-contrast information surfaces for dashboard migration later
- Tokenized color and spacing primitives for reuse across landing and protected pages
- Motion used for hierarchy only, with reduced-motion fallback

## Core Tokens
- Ink: primary text, dark surfaces, code blocks
- Deep: secondary dark action tone
- Teal: active system state, positive action, live status
- Sun: warnings, highlights, operational emphasis
- Paper: bright surface background
- Mist: muted border and surface support color

## Reusable Components
- Shell container
- Glass navigation bar
- Primary and secondary action buttons
- Section kicker label
- Panel and stat cards
- Role cards for RBAC summaries
- Code and contract blocks
- Illustration frame for product storytelling

## Landing Page Structure
1. Hero with operational positioning
2. Product illustration and live API proof
3. Platform modules grid
4. RBAC and secure workflow explanation
5. Typed contract proof area

## Future Dashboard Application
These tokens and components should be reused for:
- Protected dashboard shell
- Schedules list and detail pages
- Scraper run history and failure views
- Notifications and audit log pages
- Role-aware admin settings pages
